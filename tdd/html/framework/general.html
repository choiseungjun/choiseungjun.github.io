개요

Vue.JS란

Vue.js는 웹 개발을 단순화하고 정리하기 위해 개발된 대중적인 자바스크립트 프론트엔드 프레임워크이다.
웹 UI 개발(컴포넌트, 선언형 UI, 핫 리로딩, 타임 트래블 디버깅 등)의 아이디어를 더 접근 가능하도록 만드는데 초점을 두었으며 덜 독선적이도록 시도하고 있기 때문에 개발자들이 익히기에 더 쉽다.
코어 라이브러리는 선언형 렌더링과 컴포넌트 구성에 초점을 두며 기존 페이지에 임베드가 가능하며
라우팅, 상태 관리, 빌드 도구화와 같이 복잡한 애플리케이션에 필요한 고급 기능들은 공식적으로 유지 보수되는 지원 라이브러리와 패키지를 통해 제공한다.

  • - 프론트엔드개발 자바스크립트 프레임워크
    (* 공식홈페이지에서는 progressive framework라고 언급 -> 라이브러리 + 프레임웍)

  • - Evan You(전 구글 크리에이티브 랩 소속) 가 만듬 (https://evanyou.me/)

  • - 2014 최초 릴리즈

  • - https://vuejs.org/

현황

  • 2021 현재 국가별 관심도 (구글 트렌드)

  • 2021 현재 대한민국 관심도 (구글 트렌드)

  • 2021 현재 관심도 (state of js)

  • 2021 현재 사용 빈도 (state of js)

특징

  • - 컴포넌트 기반의 프레임워크

  • - MVVM 소프트웨어 아키텍쳐 패턴 사용


    • Model : 해당 app에서 사용되는 데이터 및 그 데이터를 처리하는 부분

    • View : 사용자에게 보여지는 UI 부분

    • ViewModel : View를 표현해주기 위한 Model이자 View를 나타내기 위해 데이터를 처리하는 부분. (MVC패턴의 C(controller)의 역할을 한다. 데이터 관리 및 액션 처리)

  • - 렌더링 된 DOM에 데이터 바인딩을 위한 HTML 기반의 템플릿 사용 (templete)
    -> Directives(v- 시작하는 속성)를 사용하여 해당 DOM을 찾음

장점

  • - 러닝커브가 낮음 (리액트나 앵귤러에 비해)

  • - 생산성이 높음

  • - 각 개발자간의 코드의 통일성이 높음

  • - 앵귤러의 장점(양방향데이터바인딩)과 리액트의 장점(Virtual DoM 기반의 렌더링)을 가지고 있음.

  • - 성능이 우수하고 빠름 (리액트나 앵귤러에 비해 좋음)

    리액트와 뷰의 렌더링 성능 비교(http://vuejs.org))

구동환경

  • - react, Angular 2와 마찬가지로 대부분의 모던 브라우저를 지원하며 인터넷 익스플로러는 9 버전부터 지원한다.
    Compatibility Note에 적혀있는 내용에 따르면 ES5 기능 중에 폴리필이 불가능한 기능을 이용하기 때문이라고 한다.

지원 라이브러리

  • - vue-cli (Command-Line-Interface) (https://cli.vuejs.org/)

    Vue 프로젝트를 손쉽게 만들 수 있도록 도와주는 커맨드라인 도구

  • - vue-router (https://router.vuejs.org/kr/)

    공식 라우터 라이브러리


    • 1. 중첩된 라우트/뷰 매핑

    • 2. 모듈화된, 컴포넌트 기반의 라우터 설정

    • 3. 라우터 파라미터, 쿼리, 와일드카드

    • 4. Vue.js의 트랜지션 시스템을 이용한 트랜지션 효과

    • 5. 세밀한 네비게이션 컨트롤

    • 6. active CSS 클래스를 자동으로 추가해주는 링크

    • 7. HTML5 히스토리 모드 또는 해시 모드(IE9에서 자동으로 폴백)

    • 8. 사용자 정의 가능한 스크롤 동작

    • 9. 설치(cmd) : vue add router

  • - vuex (상태 관리 패턴 + 라이브러리) (https://vuex.vuejs.org/kr/)


    • 1. 애플리케이션의 모든 컴포넌트에 대한 중앙 집중식 저장소 역할을 하며 예측 가능한 방식으로 상태를 변경

    • 2. 공식 devtools 확장 프로그램 (opens new window)과 통합되어 설정 시간이 필요 없는 디버깅 및 상태 스냅 샷 내보내기/가져오기와 같은 고급 기능을 제공

    • 3. 설치(cmd) : vue add vuex

  • - vue-loader (https://vue-loader-v14.vuejs.org/kr/)

    Vue 컴포넌트를 자바스크립트 모듈로 변환할 수 있는 webpack에서 사용하는 로더


    • 1. 기본적으로 ES2015를 지원합니다.

    • 2. 각 Vue 컴포넌트 마다 서로 다른 webpack 로더를 사용할 수 있습니다. 예를 들면 <style>에 Sass, <template>에 Jade로 각각 설정 가능합니다.

    • 3. <style>과 <template>에서 참조된 정적 Asset 파일을 모듈로 취급하고 webpack 로더로 처리합니다.

    • 4. 각 컴포넌트마다 지정된 CSS를 시뮬레이트 할 수 있습니다.

    • 5. 개발 중에 컴포넌트 핫 리로딩을 지원합니다.

  • - vuetify (https://vuetifyjs.com/)

    - 구글의 공식 material design을 사용할 수 있는 컴포넌트 UI 프레임워크

    - Vue환경에서 가장 활동적이며, 다양하고 완벽한 기능을 갖춘 프레임워크


    • 1. 구글 메터리얼 스펙 지원
      - 디자이너가 직접 커스터마이징 가능한 다양한 옵션 제공

    • 2. 80개이상의 시멘틱 메터리얼 디자인 컴포넌트
      - 엘리먼트 이름은 v- 시작하며 매우 직관적임
      - 모던 웹 앱에서 필요한 또는 자주 사용하는 UI/UX 제공 (그리드, 타이포, 앱 바, 카드 등)

    • 3. 빠른 속도

    • 4. 쉬운 학습
      - 공식 문서 및 샘플 제공

    • 5. Vuetify 사용 요약 (상세 설명은 공식홈페이지 별도 참조 필요)
      • 1) 설치하기
        - cdn : vuetify 홈페이지 참고
        - vue-cli 설치 : 생성된 vue 프로젝트 내에서 추가 (명령어 vue add vuetify)
        - 이후 plugins/vuetify.js 생성 및 main.js(글로벌환경)에 import되도록 작성
        import Vuetify from ‘vuetify’
        new Vue({
        router,
        store,
        render: h => h(App)
        }).$mount(‘#app’)

      • 2) 사용하기
        - Vuetify 사이트에 접속 후 사용하고자 하는 컴포넌트 선택
        - 해당 컴포넌트 우 상단의 <> 버튼을 눌러 <templete>과 <script> 코드를 확인한다.
        - 소스코드 화면에서 우 상단에 복사 버튼을 눌러 templete과 script 소스를 한번에 복사하여 사용하고자 vue 컴포넌트 파일에 붙여 넣는다.

      • 3) 컴포넌트 변경
        - 컴포넌트 API에서 props등을 이용하여 변경 가능
        - 속성값은 다양하게 추가하여 커스텀 가능.

      • 4) 기본 템플릿UI를 제공은 하지만 확장성의 문제로 많이 사용되지는 않음.

Nuxt.JS란

Vue.js 프레임워크를 기반으로 SSR(Server Side Rendering) 웹 페이지를 만들 수 있도록 해 주는 라이브러리
SEO(Search Engine Optimize) 등의 문제로 CSR(Client Side Rendering)이 아닌 SSR 웹을 구축해야 하는 경우에 유용하게 사용 가능.

  • - npx을 통해 쉽게 설치. (* vue.js의 경우 vue-cli를 통해 쉽게 설치)

    npx create-nuxt-app <project-name>

  • https://nuxtjs.org/

장점

  • 1. Serve Side Rendering

  • 2. SEO

  • 3. Layout 생성

  • 4. Router에 자동 등록

  • 5. 프로젝트 폴더 구조 고민의 가벼움

단점

  • 1. 화면 이동 시 화면 깜빡임(SSR의 단점)

  • 2. Router에 자동 등록에 불필요한 vue파일 등록

SPA(Single Page Application) :

  • - 한 개(single)의 Page로 구성된 application.
    웹 애플리케이션에 필요한 모든 정적 리소스를 최초 한번에 다운로드한다.
    그 이후 새로운 페이지 요청이 있을 경우, 페이지 갱신에 필요한 데이터만 전달받아 페이지를 갱신.


    장점)

    • 1. 페이지의 동적 작성. 변하지 않는 부분과 변하는 부분을 구분하여, 변하는 부분만 새롭게 그려주는 방식 → 쓸데없는 렌더링 비용 감소

    • 2. 클라이언트 사이드 렌더링. 템플릿 연산을 서버 측에서 클라이언트로 옮김

    • 3. 컴포넌트 단위로 개발 용이


    단점)

    • 1. 초기 load 속도가 늦음 > js, css파일들을 번들링 해서 받기 때문

    • 2. SEO가 어려움 > 번들링 된 js를 검색 봇, 크롤러들이 데이터 수집에 어려움이 있음 (요즘에는 구글 봇이 SPA도 인식을 한다고 함)

MPA(Multiple Page Application):

  • - 여러 개(multi)의 Page로 구성된 application.
    새로운 페이지를 요청할 때마다 정적 리소스가 다운로드 되고, 그에 맞춰 전체 페이지를 다시 렌더링.


    장점)

    • 1. SEO(Search Engine Optimization, 검색 엔진 최적화) 관점에 유리.
      - MPA는 완성된 형태의 HTML파일을 서버로부터 전달받는다. 따라서 검색 엔진이 페이지를 크롤링하기에 적합.


    단점)

    • 1. 새로운 페이지를 이동하면 화면이 깜빡임.
      - 새로운 페이지를 요청할 때마다 전체 페이지를 다시 렌더링 하기 때문이다.

    • 2. 프론트엔드와 백엔드가 밀접하게 연관. (Tightly coupled!)
      - 프론트엔드와 백엔드가 서로 연관이 높아 개발이 복잡.

CSR(Client Side Rendering) : SPA의 렌더링 방식

  • 장점)

    • 1. SEO 가능

    • 2. CSR에 비해 초기 로드 속도가 빠름


    단점)

    • 1. SPA에 비해 서버 성능이 더 필요함 (-> 템플릿 렌더링 비용 발생)

    • 2. 페이지 이동시 화면 깜빡임 및 중복된 렌더링 비용 발생 (→ 효율성에 대한 문제)

    • 3. SPA에 비해 초기 load속도는 빠르지만, 매 페이지마다 템플릿에 대한 새로운 load가 필요함

    • 4. 컴포넌트 단위 개발 어려움


    MPA / SPA

구조

  • - 자세한 설명은 공식홈페이지 참조.


    * Vue.js 프로젝트의 src 폴더 아래에 있던 코드들이 Nuxt에서는 루트 레벨로 이동

라우터

  • * 라우터(Router) 차이

    Nuxt 와 Vue 차이에 대한 설명
    Nuxt Vue
    항상 기능 선택적 기능
    pages 폴더 router 및 views 폴더
    자동으로 라우터 구성 (pages 폴더 구조대로) router/index.js에 직접 라우터 등록

    Ex) Nuxt.js 라우터

빌드

  • - 프로젝트를 Vue에서 빌드했을 때 dist 폴더 안에 생성되던 파일들은 Nuxt에서는 .nuxt 폴더 안으로 이동

  • - 빌드 결과물의 경우 기존 Vue.js에는 정적 배포가 가능한 파일이 생성되는 반면, Nuxt.js의 경우에는 서버 코드를 포함한 빌드 결과물들이 생성.


SEO

  • - 메타 태그 : Vue.js와 달리 별도 라이브러리 없이 메타태그 작성이 가능.

Layout

  • - Layout이라는 폴더가 추가되어, Nuxt를 통해 라우팅되는 컴포넌트를 감싸는 레이아웃을 별도로 생성 가능.
    기존 프로젝트에서 App.vue나 nested router를 사용할 때 감싸는 것과 비슷한 방식

    Ex). <router-view/> = <nuxt/>

  • - Layout은 여러 개를 생성할 수 있습니다. 특정 레이아웃을 사용할 컴포넌트 안에 layout 프로퍼티를 설정하여 layout을 사용할 수 있습니다. 아래 참고.

    <template> <!-- Your template --> </template> <script> export default { layout: '레이아웃이름' // page component definitions } </script>

LifeCycle - 참고

  • - 처음 리퀘스트가 도착하면 서버사이드에서 nuxtServeInit, middleware, validate(), asyncData(), fetch() 등의 과정을 거쳐서 렌더링 한 뒤 완성된 페이지를 리스폰스로 보낸다.
    그 이후에 페이지 이동은 페이지 갱신 없이 클라이언트에서 nuxt-link(Vue router의 router-link)를 통해 필요한 리소스만 통신으로 받아서 렌더링 한다.
    첫 요청 후 페이지 이동은 클라이언트 사이드 렌더링을 하는 것이다.
    말 그대로 server-side rendring + client-side navigation 두 가지 모두 사용
