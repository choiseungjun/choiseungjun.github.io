Intro

프로젝트를 진행 하다 보면 여러 사람이 동시에 협업 개발을 하는 경우가 대부분이다.
이에 여러 이유(버그, 개선)로 수정이 발생 할 경우 자신의 코드뿐만 아니라 다른 개발자의 작성 코드까지 읽어야(분석) 하는 경우가 발생한다.
각 개발자마다 코드 작성 스타일이나 네이밍 방식이 다르게 되면 해당 코드를 해석하는 데에 있어 많은 시간이 소요된다.
따라서 이런 소모적인 공수를 줄이고자 통일되게 UI개발 코드를 작성할 수 있도록 coding convention을 정의하고자 한다.
